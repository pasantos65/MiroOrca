# Contributing to MiroOrca

Thank you for your interest in contributing!

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your feature or fix: `git checkout -b feature/your-feature-name`
3. **Make your changes** with clear, focused commits
4. **Test** your changes locally
5. **Open a Pull Request** with a clear description of what you changed and why

## What We Welcome

- Bug fixes
- English documentation improvements
- New use-case guides in `/docs`
- UI/UX improvements to the Vue.js frontend
- Performance improvements to simulation or graph building
- New persona template presets
- Better Docker/deployment tooling

## Code Style

- **Python:** Follow PEP 8. Use `black` for formatting (`uv run black .`)
- **JavaScript/Vue:** Use ESLint with the project config
- **Comments:** Write in English. Explain *why*, not just *what*

## Reporting Bugs

Open an [Issue](../../issues) and include:
- What you were trying to do
- What you expected to happen
- What actually happened
- Your OS, Python version, Node version
- Relevant logs from `docker compose logs` or the terminal
